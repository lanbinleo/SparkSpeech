# SparkSpeech Roadmap

更新时间：2026-07-04

## 0.1.3：输入可靠性与可见反馈

0.1.3 的目标是让 SparkSpeech 从“短句可用”变成“长时间讲话也敢用”。重点不是增加 provider，而是保护录音、减少等待、让用户知道当前发生了什么。

### 默认调度

默认采用标准调度：

1. 录音开始后立即创建本地 recording session。
2. 录音过程中持续写入本地音频分段，建议每 10 到 30 秒生成一个片段。
3. 录音期间不做实时上屏。
4. 录音结束后，将完整本地音频发送给豆包 ASR。
5. ASR 完成后进入当前文本优化 Provider。
6. 录音失败、ASR 失败或优化失败时，保留本地音频和历史记录。

完整本地音频是主流程 ASR 来源，也负责安全保存、排查和手动重新转写。

### 时间轴数据

0.1.3 需要保存音频分段数据，方便恢复、排查和后续升级：

- 音频分段：`start_ms`、`end_ms`、`path`、`status`
- 处理来源：`final_full_asr`

### 音频保护

- 录音 session 应有 manifest 文件，记录 session id、开始时间、当前状态和分段列表。
- 每个音频分段写入完成后更新 manifest。
- 正常结束录音时，将分段合成为最终 WAV，并保存历史记录。
- App 启动时扫描未完成 session。
- 若存在未完成 session，生成一条“未完成录音”历史记录，允许试听、打开文件夹或重新转写。
- 录音失败、网络失败、优化失败都不得删除已保存音频。

### 每日备份

- App 启动时检查当天是否已有备份。
- 若没有，创建 `backups/YYYY-MM-DD.tar.gz`。
- 默认备份内容：
  - `settings.json`
  - `prompts.json`
  - `records.json`
  - 最近日志
  - 未完成录音 session manifest
- 备份默认保留 7 天。
- 音频文件主要通过分段保存机制保护；后续可以增加“备份包包含最近音频”的设置。

### Overlay 设计

Overlay 保持一个横向状态条，不做实时上屏。

横向状态条尽量使用 icon，少用说明文字：

- 正在录音：录音机 icon。
- 录音中：右侧显示已录制时长。
- 最近分段已保存：绿色勾 icon，短暂显示后回到录音状态。
- 正式转写中：圆环按已处理音频时长 / 总时长显示。
- 文本优化中：圆环按流式输出字符数显示。

优化阶段的总量只能作为参考值：

- 字符进度需要文本优化 Provider 使用 streaming。
- 右侧显示 `已输出字符数 / 约 ASR 字符数`。
- 进度条在模型结束前最多显示到 92% 到 95%。
- 收到结束信号后平滑到 100%。
- 若输出超过参考值，数字继续增长，进度条保持接近完成状态。

### 非目标

0.1.3 暂不实现“只转写中断区间并拼成最终文本”的激进调度。当前只保留时间轴数据，为后续验证做准备。

0.1.3 暂不引入新的 ASR provider。豆包仍是唯一 ASR provider。

0.1.3 支持多个 OpenAI-compatible 文本优化 Provider：OpenRouter、DeepSeek 和自定义 Provider。

## 0.1.4 候选方向

- 历史记录全文搜索。
- 按状态和日期筛选历史记录。
- 编辑最终文本后再复制或粘贴。
- 重新优化时临时选择模型或整理模板。
- 日志页增加复制诊断信息。

## 0.2.x 候选方向

- 从 JSON 迁移到 SQLite。
- 支持大量历史记录下的快速搜索和分页。
- 支持 MP3、M4A、FLAC 导入。
- 用户词典分组和导入导出。
- 更完整的托盘菜单。
